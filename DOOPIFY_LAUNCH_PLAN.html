Phase 1 — Foundation

Blocks everything else. Nothing in phases 2–8 can happen until this is done. Start here immediately.

Environment Setup
CriticalEst: 1 day
›

Set up the core infrastructure everything else will run on.

Security Requirements

Current security posture: zero. Any URL in the admin is publicly accessible right now. No auth, no protection, no rate limiting. Every item below must be addressed before you handle real customer or payment data.
Admin Route Protection
All admin routes must require a valid session. Unauthenticated requests redirect to /login. Implement in Next.js middleware.ts at the root — runs on Edge before any page renders.
middleware.ts + JWT verify
Password Hashing
Never store plaintext passwords. Hash with bcrypt (cost factor 12). Compare hashes on login — never compare raw strings.
bcryptjs · cost 12
JWT in httpOnly Cookie
Store the JWT token in a httpOnly; Secure; SameSite=Strict cookie — not localStorage. httpOnly prevents JavaScript from reading it, blocking XSS token theft entirely.
Set-Cookie: token=...; httpOnly; Secure
Rate Limiting
Limit login endpoint to 5 attempts / 15 minutes per IP. Use @upstash/ratelimit with Redis for stateless rate limiting that works on Vercel Edge.
@upstash/ratelimit + Upstash Redis
Input Validation (Zod)
Validate and parse every API request body before it touches Prisma. Reject unknown fields, enforce types, validate string lengths. Return structured 400 errors for invalid input — never expose internal errors.
Zod schemas on every POST/PATCH handler
Stripe Webhook Signatures
Verify every incoming Stripe webhook with the signing secret. Unsigned events must be rejected with 400. This prevents anyone from forging order creation events.
stripe.webhooks.constructEvent()
CORS Configuration
Lock API routes to your own domain. Public storefront API routes can accept requests from the storefront origin only. Admin routes should never be callable from external origins.
next.config: headers() → CORS
Security Headers
Add X-Frame-Options: DENY, X-Content-Type-Options: nosniff, Referrer-Policy, and Content-Security-Policy headers to all responses. Prevents clickjacking and MIME sniffing.
next.config: headers() config block
SQL Injection Prevention
Prisma's parameterized queries prevent SQL injection by default. Never use raw queries with string interpolation. If you must use prisma.$queryRaw, use tagged templates — not string concatenation.
Prisma ORM handles this — never concat SQL
Environment Variables
Never commit .env.local. Rotate secrets if ever exposed in git history. Use Vercel's encrypted environment variable storage. Prefix public vars with NEXT_PUBLIC_ — treat all others as server-only.
.gitignore + Vercel dashboard
Role-Based Access
Admin users have roles: OWNER, STAFF, VIEWER. OWNER can delete products and manage users. STAFF can manage orders and products. VIEWER is read-only. Enforce in API middleware — never just hide in UI.
requireRole() middleware helper
Inventory Race Conditions
When two people buy the last item simultaneously, both can pass the inventory check. Use Prisma's atomic decrement combined with a where: { inventory: { gt: 0 } } clause to prevent overselling. If the update returns 0 rows, the item is sold out — cancel that checkout.
Prisma atomic update · conditional where

Suggested Features to Add

⬡
Collections / Smart Filters
Let admin create collections with manual or automated rules (e.g. "all products tagged 'sale' under $50"). Storefront renders collection pages dynamically.
⬡
Abandoned Cart Recovery
When a user starts checkout but doesn't complete — send a timed email (1hr, 24hr) with a link back to their cart. Resend + Stripe incomplete PaymentIntents makes this straightforward.
⬡
Product Reviews
Simple star rating + text review tied to verified purchases. Visible on product detail pages. Moderation queue in admin. Boosts conversion significantly.
⬡
Inventory Alerts
Notify admin by email when a variant's inventory drops below a configured threshold. Configurable per-product or globally in Settings. Prevents stockouts.
⬡
Storefront Search
Full-text product search using Prisma full-text search (PostgreSQL). Add a search bar to the storefront header. Optional: add Algolia for faceted search at scale.
⬡
Tax Calculation
Integrate Stripe Tax or TaxJar to auto-calculate sales tax by shipping address. Without this, you either charge no tax (risky) or charge flat tax (inaccurate).
⬡
Gift Cards
Issue gift card codes via Stripe (or custom DB codes). Redeemable at checkout like a discount. Tracked against a balance in the DB. Popular and high-margin product.
⬡
Real-Time Inventory Updates
Show "Only 3 left" live on product pages using SWR polling or Next.js Server-Sent Events. Updates without page refresh when inventory changes from another purchase.
⬡
Admin Activity Log
Log every admin action (who created/edited/deleted what, when) to an audit table. Critical for team accountability and debugging production issues.
⬡
Bulk Operations
Select multiple products → bulk price update, bulk status change, bulk tag assignment. The admin UI already has checkbox infrastructure — just needs the API endpoints.
⬡
CSV Import / Export
Import products from a CSV (for migration from Shopify or spreadsheets). Export orders, customers, products as CSV for accounting and reporting.
⬡
Webhook Outbound Events
Let store owners configure webhook endpoints to call on order events (order.created, fulfillment.updated). Foundation for a future app/plugin ecosystem.
